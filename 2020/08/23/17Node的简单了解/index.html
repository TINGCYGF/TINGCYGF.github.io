Node简单了解

发表于 2020-08-23 更新于 2020-10-20 分类于 Node

Node是什么？

它是一个将多种技术组合起来让Javascript也能调用的系统接口，开发后端应用。

Node.js的技术

  1. V8引擎
  2. libuv
  3. C/C++实现的c-ares、http-parser、OpemSSL、zlib等库
  4. 了解更多：https://github.com/yjhjstz/deep-into-node
Node.js API(http模块、fs模块、stram模块)等
Node.js blindings(让js与C/C++通信) C/C++插件(自定义其他能力)
js引擎V8 跨平台的异步/O通信能力libuv DNS解析c-ares 加密解密OpenSSL 其他zlib等库

bindings

背景：C/C++实现一个http_parser库很高效，但是只会写JS。

所以bindings当JS与C/C++的桥梁

  1. Node.js用C++对http_parser进行封装，封装文件叫http_parser_bindings.cpp
  2. Node.js提供编译工具编译为.node文件
  3. 这样就可以JS调用C++库，Node.js提供很多binding，所以+s。就是bindings

JS调用C++

官方示例：http://nodejs.cn/api/addons.html#addons_function_arguments

C++调用JS回调

官方示例：http://nodejs.cn/api/addons.html#addons_callbacks

V8引擎

功能：

  1. 将JS源代码变成本地代码并执行
  2. 维护调用栈，确保JS函数的执行顺序
  3. 内存管理，为手游对象分配内存
  4. 垃圾回收，重复利用无用的内存
  5. 实现JS的标准库

注意点：

  1. V8不提供DOM API
  2. V8执行JS是单线程
  3. 可以开启两个线程分别不同JS
  4. V8是多线程的：如垃圾回收为单独线程
  5. 自带event loop 但Node.js基于libuv自己做了一个

libuv

背景：

  1. FreeBSD系统有kqueue、Linux系统上有epoll、Windows系统有IOCP
  2. Ryan（作者）为了实现跨平台的异步I/O库，开始写libuv
  3. libuv会根据系统自动选择合适的方案

功能：用于TCP/UDP/DNS文件的异步操作

Event Loop

什么是Event

  1. 内部的计算器到期了事件
  2. 外部的文件可以读取、读取出错事件
  3. socket有内容了、关闭了告诉事件

什么是Loop

  1. loop就是循环
  2. 事件有优先级的所以处理起来也是分先后的，Node.js需要顺序论询每种事件，所以往往是循环的

Event Loop

  1. 操作系统可以触发事件，js可以处理事件
  2. Event Loop就是对事件处理的顺序管理

顺序示意图翻译：https://juejin.im/post/6844903582538399752

重点阶段

timers阶段：检查计时器

poll轮询阶段：当轮询没事情处理后会停止在poll阶段

check检查setlmmediate回调

注意：大部分事件停在poll，文件、网络请求都在poll阶段
