Vue文档总结笔记

发表于 2020-03-08 更新于 2020-04-17 分类于 Vue

本博客总结了VUE框架的基本使用方法与部分原理解析，可搭配Vue官方文档观看。

  • Vue是一个渐进式的框架
    • 渐进式意味着你可以将Vue作为你应用的一部分嵌入其中
    • 与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动
    • 比如Core+Vue-router+Vuex可以满足需求
  • Vue的特点和web开发中常用的高机功能
    • 解耦视图和数据
    • 可复用的组件
    • 前端路由技术
    • 状态管理
    • 虚拟DOM
  • 学习Vue的前提
    • 具备HETML、CSS、JavaScript基础

Vue中的MVVm

  • Mvvm定义:MVVM是Model-View-ViewModel的简写。即模型-视图-视图模型。
    • 【模型】指的是后端传递的数据。
    • 【视图】指的是所看到的页面。
    • 【视图模型】mvvm模式的核心，它是连接view和model的桥梁。它有两个方向：一是将【模型】转化成【视图】，即将后端传递的数据转化成所看到的页面。实现的方式是：数据绑定。二是将【视图】转化成【模型】，即将所看到的页面转化成后端的数据。实现的方式是：DOM 事件监听。这两个方向都实现的，我们称之为数据的双向绑定。
    • 总结：在MVVM的框架下视图和模型是不能直接通信的。它们通过ViewModel来通信，ViewModel通常要实现一个observer观察者，当数据发生变化，ViewModel能够监听到数据的这种变化，然后通知到对应的视图做自动更新，而当用户操作视图，ViewModel也能监听到视图的变化，然后通知数据做改动，这实际上就实现了数据的双向绑定。并且MVVM中的View 和 ViewModel可以互相通信。

Vue实例中传入的options

v-if判断语句

Vue的条件指令可以根据表达式的值在DOM中渲染或组件

原理：v-if后面的条件为false时对应的元素以及子元素不进行渲染

也就是根本不会有对应的标签出现在DOM中

v-if=”massage”//判断是否
v-else-if=”massage”//先判断排除上在判断
v-else//最后剩余，不用=对象名

虚拟DOM渲染复用与key

vue在进行DOM渲染时，出于性能考虑，会尽可能复用，而不是重新创建元素。

解决方案：添加key并保证不同即可避免复用

v-show与v-if的区别

v-show与v-if都可以决定一个元素是否渲染

  • 区别：
    • v-if当条件为false时，不会有对应的元素在DOM中
    • v-show条件为false时，仅是将元素display属性设置为none
  • 开发中如何选择
    • 当显示与隐藏频繁时用v-show
    • 当只有一次切换时用v-if

v-for遍历数组

语法格式：v-for=”(value, key, index) in info

依此取出内容和属性名和索引值

v-for绑定key与不绑定key的区别

官方推荐使用v-for时给对应元素或组件添加一个key属性

vue中和diff算法有关系

不绑定时：当插入元素会重新排列，性能低

绑定key要一对应（不能绑定index）：添加唯一标识，插入时可以找到节点插入

key的作用为了高效更新虚拟DOM

哪些数组方法是响应式的

Vue是响应式的，所以当数据改变时，Vue会自动检测并更新到DOm

所以其中总结有一些方法是响应式的

  • push()增加元素（元素，元素）
  • pop()删除最后
  • shift()删除第一个元素
  • unshift()最前面添加元素
  • splice()删除/插入/替换
    • （第一个元素表示第几个开始删除，第二删除几个，）
    • 没有第二个就从那里后面全部删除
    • 第二个元素表示替换几个元素
    • 后面更上替换元素
    • 最后跟上添加元素
  • sort()
  • reverse()反转
  • Vue.set(this.要修改的对象，索引值修改的地方，修改后的值)
